Building Your Ghost Theme

A step-by-step guide to get you from zero to building your theme with Claude Code.

The big picture

You're going to install a few tools on your Mac, then use an AI assistant called Claude Code to help you turn your Figma designs into a working Ghost theme. You'll talk to Claude in plain English — describe what you want, and it writes the code.

Everything you build gets saved with version control (think: unlimited undo history). You can experiment freely because you can always go back to a version that was working.

You and your collaborator will share the project through GitHub — like Google Docs for code. You both see each other's changes.

Part 1: Accounts You Need

Before installing anything, create these two accounts. Both are free to start.

1 Create a Claude account

Go to claude.ai and sign up.

Then subscribe to the Pro plan ($20/month) at claude.ai/pricing.

Why: Claude Code is the AI that helps you write code. The Pro plan gives you access to it. Think of it like hiring a developer who works inside your terminal.
2 Create a GitHub account

Go to github.com and sign up. Free account is fine.

After you create your account, tell your collaborator your username so they can give you access to the shared project.

Why: GitHub is where your project lives online. It's like a shared Dropbox folder specifically built for code — but with a full history of every change anyone ever made. You and your collaborator both work on the same project through GitHub.

Part 2: Install Your Tools

These steps happen in the terminal — a text-based app where you type commands instead of clicking buttons. It looks old-school, but it's incredibly powerful once you get used to it.

First: Download a terminal app

3 Download Warp

Go to warp.dev and download the app. Install it like any other Mac app (drag to Applications).

Open Warp. It'll ask you to create a Warp account — follow the prompts.

Why Warp instead of the built-in Terminal? Warp is a modern terminal designed to be less intimidating. Commands and their results are visually separated into blocks (like messages in a chat). It has autocomplete that shows what commands do. Once you get comfortable, you can switch to any terminal you want — but Warp is the friendliest starting point.

Next: Run the setup script

Your collaborator sent you a file called setup.sh along with this guide. This single file installs everything else you need.

4 Move setup.sh to your home folder

Open Finder. Find setup.sh (probably in Downloads or wherever you received it). Move it to your home folder.

Your home folder is the one with the house icon in Finder's sidebar. It has your name on it. The path is something like /Users/luis/.

5 Run the setup script

In Warp, type this command and press Enter:

$ bash ~/setup.sh

"Running" a command means: you type it into the terminal and press Enter. The computer then executes that instruction.

This command means: "Use bash (a program that reads scripts) to run the file called setup.sh in my home folder (~/ is shorthand for your home folder)."

The script will take a few minutes. It prints what it's doing at each step. You may be asked for your Mac password partway through — this is normal. Type your password (it won't show the characters — that's a security feature, not a bug) and press Enter.

What this installs:
  • Homebrew — a tool that installs other tools (like an app store for the terminal)
  • Git — version control (your "save point" system)
  • GitHub CLI — connects your terminal to GitHub
  • Node.js — a behind-the-scenes tool that Ghost themes need
  • Claude Code — your AI coding partner
After the script finishes, close Warp completely (Cmd+Q) and reopen it. This is necessary so the terminal recognizes the new tools you just installed.

Part 3: Connect to GitHub

Now that your tools are installed, you need to connect your terminal to your GitHub account. This is a one-time setup.

6 Log into GitHub from the terminal

In Warp, type this command and press Enter:

$ gh auth login

It will ask you a series of questions. Here's exactly what to choose:

It asks...You choose...
Where do you use GitHub?GitHub.com
What is your preferred protocol?HTTPS
How would you like to authenticate?Login with a web browser

It shows you a code and opens your web browser. Paste the code into the browser, click Authorize, and you're done.

What just happened: Your terminal can now talk to GitHub. When you "push" code (send it to GitHub) or "pull" code (download changes someone else made), it knows who you are. You won't need to log in again.

Part 4: Get Your Project

Your collaborator has already created the project on GitHub and set it up with everything you need. Now you'll download a copy to your Mac.

7 Clone the project

In Warp, type these commands one at a time, pressing Enter after each:

# Download the project to your Mac
$ gh repo clone whale/ghost-theme-luis ~/ghost-theme-luis

# Move into the project folder
$ cd ~/ghost-theme-luis

"Clone" means download a copy of the project from GitHub to your computer. The ~/ghost-theme-luis part tells it where to put it — in a folder called ghost-theme-luis inside your home folder.

"cd" stands for "change directory" — it's like double-clicking a folder in Finder. You're telling the terminal "I want to be inside this folder now."

What you just downloaded:
  • Your starter theme files (the skeleton of a Ghost theme — ready to customize)
  • A reference guide that teaches Claude how Ghost themes work
  • Instructions (CLAUDE.md) that tell Claude how to work with you
8 Download the reference themes

Run the setup script one more time — now that the project folder exists, it'll download 20+ professional Ghost themes that Claude will study as examples:

$ bash ~/setup.sh

This adds a reference/ folder inside your project with all the official Ghost themes. You don't need to read these — they're for Claude to learn from.

Part 5: Start Building

This is the fun part. You're going to talk to Claude in plain English and build your theme together.

9 Launch Claude Code

Make sure you're in your project folder, then start Claude:

$ cd ~/ghost-theme-luis
$ claude

First time only: It opens a browser window for you to log in with your Claude account. Do that, and it drops you back into the terminal.

You're now inside Claude Code. You'll see a prompt where you can type in plain English. Try saying something like:

You: What files are in my project? Give me a quick overview.

Claude reads your project, understands the structure, and responds. It can see all the files, read the reference themes, and use the documentation — all automatically.

10 Start building from your Figma design

Share your Figma design with Claude by describing what you see, or share a screenshot. Then tell it what to build:

You: I want my homepage to have a large title at the top, then a list of my posts below with just the title and date. The font should be Inter. Here's what it looks like in my Figma design: [describe or paste screenshot]

Claude will write the HTML and CSS, explain what it did, and save your progress automatically.

Part 6: See Your Theme on Ghost

11 Upload your theme to Ghost

When you want to see how your theme looks on your actual Ghost site:

  1. Tell Claude: "Package my theme for upload"
  2. Claude creates a .zip file and tells you where it is
  3. Open your Ghost Admin panel (your-site.ghost.io/ghost/)
  4. Go to Settings → Design → Change theme → Upload theme
  5. Choose the .zip file Claude created
  6. Visit your site to see it live
Why the zip step? Ghost themes are uploaded as compressed packages. Claude packages everything in your dev/luis/ folder into a single file that Ghost understands. Later, we can set up automatic deployment so this happens every time you save — but for now, the manual upload keeps things simple.

Daily Workflow

Once everything is set up, this is your routine:

  1. Open Warp
  2. Type cd ~/ghost-theme-luis && claude and press Enter
  3. Tell Claude what you want to work on
  4. When you want to see your changes on Ghost, say "package my theme" and upload the zip
  5. When you're done for the day, say "save my progress and push to GitHub"

Key Concepts (in plain language)

Terminal

A text-based way to control your computer. Instead of clicking icons, you type short commands. Looks intimidating at first, but you'll mostly just type cd (to go to a folder) and claude (to start your AI helper). Claude handles the rest.

Git & GitHub

Git is your save-point system. Every time you (or Claude) "commit," it takes a snapshot of your entire project. You can go back to any snapshot at any time. Nothing is ever truly lost.

GitHub is where those snapshots live online. It's how you and your collaborator share work — like a Google Doc, but for code. You both push your changes up and pull each other's changes down.

Ghost Theme

A Ghost theme is a collection of files that tell your Ghost website how to look. There are three types of files:

  • .hbs files (Handlebars templates) — the structure of each page. Like wireframes that say "put the title here, the date there, the content below."
  • .css files — the styling. Colors, fonts, spacing, layout. This is where your Figma design becomes real.
  • .js files — behavior (optional). Things like "load more posts when I scroll down" or "show a menu when I click the hamburger icon."

Claude Code

An AI that lives in your terminal. You describe what you want in plain English, it writes the code. It can read all your project files, study the reference themes, and make changes — but it always explains what it did and asks before doing anything risky. Think of it as a developer you're creative-directing.
